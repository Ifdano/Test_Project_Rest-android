## Получение данных с сервера, с помощью Retrofit. А также, хранение данных в базе данных[Sqlite], использование RxJava/RxAndroid и загрузка изображения Picasso.
## Обобщенно: Retrofit + RxJava/RxAndroid + Sqlite + Picasso + ListView + Fragment + Singleton[Design Patterns].
### Подробнее.

### 1]Общее меню, где можно выбрать интересующие нас данные: текущая погода, шутка или информация о пользователе Github.

### 2] После выбора интересующих нас данных, мы переходим на экран для получения самих данных. 
###    Если выбрана текущая погода, то нужно ввести название города и получить данные с сервера[https://openweathermap.org/current], которые будут добавлены в базу данных, а оттуда в общий список. Список состоит лишь из краткой информации, а чтобы получить более раскрытую/обширную информацию - нужно выбрать нужные данные из списка.  
![screenshot_20190116-235830 u8y70](https://user-images.githubusercontent.com/15383481/51281094-deae6400-19fa-11e9-9f98-54f351c98238.png)
![screenshot_20190116-235916 h4xqh](https://user-images.githubusercontent.com/15383481/51281095-df46fa80-19fa-11e9-83dd-e46624cc4db2.png)
### Для получения шуток с сервера[https://github.com/15Dkatz/official_joke_api] - ничего не нужно вводить. А для получения данных пользователя Github с сервера[например, https://api.github.com/users/ifdano] - нужно ввести имя пользователя[username].

### 3] После выбора нужных нам данных из списка - мы переходим на экран с раскрытой/обширной информацией. 
### Для погоды указаны: температура, влажность воздуха, скорость ветра и другие данные.
![screenshot_20190116-235838 damvd](https://user-images.githubusercontent.com/15383481/51281275-5d0b0600-19fb-11e9-9af5-31f92fb0267f.png)
![screenshot_20190116-235933 skqdw](https://user-images.githubusercontent.com/15383481/51281277-5d0b0600-19fb-11e9-9ccb-3443bcbb2b8b.png)
### Для пользователей Github указаны: имя, дата создания профиля, ссылка на профиль и показана аватарка.
### Для шуток указаны: начало шутки[setup] и конец[punchline].
### После получения раскрытой информации - мы можем изменить данные[UPDATE] или удалить[DELETE], сделав соответствующие запросы к базе данных. 

### 4] Для удаления данных, нужно дважды нажать на кнопку удаления DELETE. В этом случае, данные удалятся из базы данных.
###    Для изменения данных, нужно нажать на кнопку UPDATE. В этом случае, мы перейдем на экран для изменения данных. Все данные будут показаны в полях. После завершения - нужно подтвердить запрос[SUBMIT] на изменение данных[в базе данных]. Если изменять ничего не нужно, то можно просто вернуться обратно, нажав на CANCEL.
